CMC Filters Engine v2.3

Custom Webflow CMS filter system — no Finsweet, JS with n4-* attributes.

Replaces Finsweet's fs-cmsfilter with a JS engine built for CMC's multi-level filter hierarchy. Supports up to 5 visible levels with cascading show/hide, availability muting, and Carlton's fork logic (conditional Level 3/4 based on Level 2 choice). Each level acts as a gate — AND between levels, OR within a level.

data-* Attributes

Used for structural cascade (show/hide groups, counts, labels). These come from the Webflow DOM — the JS reads them as-is.

Naming convention: data-filters-* (plural) on structural containers and display receivers (level wrappers, count spans, label receivers). data-filter-* (singular) on individual items, list wraps, and group/fork identifiers. Both patterns are set in Webflow Designer and used consistently throughout the engine.

Matching Logic

Category (Level 1)

// Radio n4-list-value is a slug; product category <p> textContent is a display name.
// toSlug() bridges the two.
const selectedCategory = radio.getAttribute('n4-list-value'); // e.g. "cmc-lifts"
const productCategory  = categoryEl.textContent.trim();       // e.g. "CMC Lifts"
const categoryMatch = !selectedCategory || toSlug(productCategory) === selectedCategory;

Filters (Level 2+) — AND between levels, OR within level

// Each level is collected separately:
const level2Filters = getCheckedLevel(2); // e.g. ["carlton-type-wood-chipper"]
const level3Filters = getCheckedLevel(3); // e.g. ["carlton-wood-chipper-cutting-system-disc"]
const level4Filters = getCheckedLevel(4); // e.g. ["carlton-wood-chipper-capacity-12-inch"]

// Product must pass EVERY active level (AND), matching ANY value within it (OR):
const filterMatch = [level2, level3, level4, level5].every(lvl =>
  lvl.length === 0 || lvl.some(val => productFilters.includes(val))
);
Gate logic: Selecting Working Height 40ft and Application = Contractor Rental shows only products that have both filters — not one or the other. Multiple values within the same level still use OR (e.g. checking 40ft + 50ft shows either).

Cascade reset rules

// Level 1 change → resetLevels() → hides + unchecks levels 2–5
// Level 2 change → resetLevel(3), resetLevel(4), resetLevel(5) → then show Level 3
// Level 3 change → resetLevel(4), resetLevel(5) → then show Level 4

Level 1 → Level 2 cascade

// Radio display value → slug → match data-filter-list-id on Level 2 groups
const categorySlug = toSlug(radio.getAttribute('n4-list-value')); // "cmc-lifts"
// Only the group with data-filter-list-id="cmc-lifts" becomes visible

Level 2 → Level 3 cascade

// Level 3 shows when at least one Level 2 checkbox is checked
// Level 3 (and 4, 5) are reset first on every Level 2 change
// showLevel3(categorySlug) re-runs with fresh state

Carlton Fork Logic

Carlton only. All other categories use a flat checkbox approach. Carlton Level 2 is a fork: checking Stump Grinder or Wood Chipper determines which Level 3 (and future Level 4) filters appear.

How fork detection works

The engine detects a fork automatically by checking whether any Level 3 [data-filter-dropdown="level-3-filters-item"] elements inside the active category group have a non-empty data-filter-parent-category attribute. No hardcoded category names. 

const isFork = !!activeGroup.querySelector(
  '[data-filter-dropdown="level-3-filters-item"]' +
  '[data-filter-parent-category]:not([data-filter-parent-category=""])'
);

Level 2 mutual exclusion

When Stump Grinder is checked, Wood Chipper gets muted (opacity 0.5, pointer-events none) and vice versa. Implemented in updateLevel2ForkMuting() which runs after updateLevel2Availability() on every Level 2 checkbox change.

Level 3 item / label visibility

Each Level 3 item has data-filter-parent-category set to the Level 2 slug it belongs to (e.g. carlton-type-stump-grinder). showLevel3() hides items whose parent category doesn't match the currently checked Level 2 value. Label elements work the same way. 

// Items with data-filter-parent-category set in Webflow CMS:
//   carlton-stump-grinder-drive-type-tracks  → data-filter-parent-category="carlton-type-stump-grinder"
//   carlton-wood-chipper-cutting-system-disc → data-filter-parent-category="carlton-type-wood-chipper"

// showLevel3() shows only items matching the active fork parent
item.style.display = (parentCat === selectedParent) ? '' : 'none';

Level 3 availability muting with fork

updateLevel3Availability() skips items that are already fork-hidden (display: none) so fork visibility and availability muting don't conflict.

User Experience

Written for designers. No code — just behavior.

The filter system is a progressive multi-step funnel. Each step narrows the product list. Steps appear one at a time as the user makes selections — the page never shows all filters at once. Selected filters are reflected as tag chips below the dropdowns.

Step-by-step Flow

Step 1 — Equipment Type (always visible)

A radio button group. The user picks one category: CMC Lifts, Lift Carriers, Trailers, or Carlton. Only one can be selected at a time. Selecting a category immediately:

  • Filters the product list to show only that category
  • Reveals Step 2 with options relevant to that category
  • Adds a tag chip for the selected category

Step 2 — Primary Filter (appears after Step 1)

A checkbox group. The user can select one or more values. Checking any box:

  • Further narrows the product list
  • Reveals Step 3
  • Adds a tag chip for each checked value

The label above Step 2 changes per category — e.g. Application for CMC Lifts, Compatible with for Lift Carriers, Brand for Trailers, Carlton Type for Carlton.

Carlton only — Step 2 is a fork. Stump Grinder and Wood Chipper are mutually exclusive. Checking one immediately mutes the other. The entire Step 3 and Step 4 content changes depending on which branch the user is on. See Carlton notes below.

Step 3 — Secondary Filter (appears after Step 2)

Another checkbox group. Appears only when at least one Step 2 value is checked. Selecting a value reveals Step 4 (if the category has one).

Step 4 — Tertiary Filter (appears after Step 3)

Another checkbox group. Appears only when at least one Step 3 value is checked. For CMC Lifts, selecting a value here reveals Step 5. Lift Carriers never has a Step 4 — the level is skipped entirely.

Step 5 — Lift Carrier Compatible (CMC Lifts only)

Appears only for CMC Lifts, only after a Step 4 selection. Has two values: Yes and No. No other category uses Step 5 — the container exists in the DOM but stays permanently hidden for Carlton, Trailers, and Lift Carriers.

Muting Behavior

Muted items appear at 50% opacity and are not clickable. Muting communicates "this filter exists but no products match it given your current selections." It is not a disabled state — it is a contextual hint.

Step 2 muting — static availability

Step 2 items are muted based on the entire selected category. If no product in e.g. CMC Lifts has the filter "Contractor & Rental", that checkbox is muted from the start. This check runs once when the category is selected and again after any Step 2 change.

Step 3 muting — dynamic availability

Step 3 items are muted based on currently visible products — not the full category. As the user checks Step 2 boxes and the product list narrows, Step 3 availability updates in real time. A Step 3 value is muted if no currently-visible product has that filter.

Step 4 & 5 muting — same as Step 3

Same dynamic logic — muted if no currently visible product has that filter value. Fork-hidden items (Carlton branch items that don't belong to the active branch) are never affected by muting logic — they remain hidden, not muted.

Carlton fork muting — mutual exclusion

In Carlton Step 2, Stump Grinder and Wood Chipper are mutually exclusive. Once the user checks one, the other becomes muted and unclickable. This prevents impossible filter combinations (a product cannot be both). If the user removes the checked value (via tag or by unchecking), both options become available again.

Filter Tags

A strip of tag chips appears below the filter dropdowns whenever any filter is active. Each chip shows the human-readable label of the selected filter (e.g. Arbor, 40–60 ft). The strip is hidden when no filters are active.

Tag rules

  • One tag per selection — each checked checkbox or selected radio gets its own chip
  • Step 1 has only one tag — selecting a new category replaces the previous tag
  • Tags appear left to right in the order the user selected them (L1 first, L5 last)
  • Clicking ✕ on a tag removes that filter and clears all downstream steps and their tags
  • Manually unchecking a checkbox also removes its tag from the strip
Example: User has selected CMC Lifts → Arbor → 40–60 ft → 23–30 ft. Four tags are visible. Clicking ✕ on "Arbor" removes the Arbor tag, clears Steps 3/4/5, and re-runs the product filter. Steps 3–5 disappear. The product list updates immediately.

Cascade Reset Rules

Every time a higher-level filter changes, all downstream steps are fully reset — hidden, unchecked, and their tags removed. This prevents impossible combinations like having a Working Height selected but no Application selected.

User actionWhat gets reset
Changes Step 1 (new category)Steps 2, 3, 4, 5 — all hidden and unchecked
Checks or unchecks any Step 2 boxSteps 3, 4, 5 — hidden and unchecked
Checks or unchecks any Step 3 boxSteps 4, 5 — hidden and unchecked
Checks or unchecks any Step 4 boxStep 5 — hidden and unchecked
Checks or unchecks any Step 5 boxNothing — Step 5 is the last level
Clicks ✕ on a Step 1 tagSteps 2, 3, 4, 5 — all hidden and unchecked
Clicks ✕ on a Step 2 tagSteps 3, 4, 5 — hidden and unchecked
Clicks ✕ on a Step 3 tagSteps 4, 5 — hidden and unchecked
Clicks ✕ on a Step 4 tagStep 5 — hidden and unchecked

The product list re-filters after every reset. If all steps are cleared, all products in the selected category are shown. If Step 1 is also cleared, all products across all categories are shown.

Per-category Notes

CMC Lifts 5 steps

The deepest category — all five filter steps are available. Step 5 (Lift Carrier Compatible: Yes/No) is exclusive to this category and appears only after a Step 4 selection.

Lift Carriers 2 steps

Steps 1–3 only. Step 4 is permanently hidden — Lift Carriers has no Level 4 filters. The Level 4 group exists in the DOM but is empty, so the container stays hidden. Step 5 never appears.

Trailers 3 steps

Steps 1–4. Step 5 never appears.

Carlton Fork

Steps 1–4, but Step 2 is a fork point. The user chooses either Stump Grinder or Wood Chipper — not both. This choice determines the content of Steps 3 and 4 entirely:
