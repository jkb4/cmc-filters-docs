CMC Filters Engine v1.7

Custom Webflow CMS filter system — no Finsweet, pure JS with n4-* attributes.

Replaces Finsweet's fs-cmsfilter with a hand-written engine tuned for CMC's multi-level filter hierarchy. Supports up to 5 visible levels with cascading show/hide, availability muting, and Carlton's fork logic (conditional Level 3/4 based on Level 2 choice). Each level acts as a gate — AND between levels, OR within a level. All logic lives in code/filters_engine.js.

Matching Logic

Category (Level 1)

// Radio n4-list-value vs product category <p> textContent
const selectedCategory = radio.getAttribute('n4-list-value'); // e.g. "Carlton"
const productCategory  = categoryEl.textContent.trim();       // e.g. "Carlton"
const categoryMatch = !selectedCategory || productCategory === selectedCategory;

Filters (Level 2+) — AND between levels, OR within level

// Each level is collected separately:
const level2Filters = getCheckedLevel(2); // e.g. ["carlton-type-wood-chipper"]
const level3Filters = getCheckedLevel(3); // e.g. ["carlton-wood-chipper-cutting-system-disc"]
const level4Filters = getCheckedLevel(4); // e.g. ["carlton-wood-chipper-capacity-12-inch"]

// Product must pass EVERY active level (AND), matching ANY value within it (OR):
const filterMatch = [level2, level3, level4, level5].every(lvl =>
  lvl.length === 0 || lvl.some(val => productFilters.includes(val))
);
Gate logic: Selecting Working Height 40ft and Application = Contractor Rental shows only products that have both filters — not one or the other. Multiple values within the same level still use OR (e.g. checking 40ft + 50ft shows either).

Cascade reset rules

// Level 1 change → resetLevels() → hides + unchecks levels 2–5
// Level 2 change → resetLevel(3), resetLevel(4), resetLevel(5) → then show Level 3
// Level 3 change → resetLevel(4), resetLevel(5) → then show Level 4

// Radio display value → slug → match data-filter-list-id on Level 2 groups

// Only the group with data-filter-list-id="cmc-lifts" becomes visible

// Level 3 shows when at least one Level 2 checkbox is checked
// Level 3 (and 4, 5) are reset first on every Level 2 change
// showLevel3(categorySlug) re-runs with fresh state

Carlton Fork Logic

Carlton only. All other categories use a flat checkbox approach. Carlton Level 2 is a fork: checking Stump Grinder or Wood Chipper determines which Level 3 (and future Level 4) filters appear.

How fork detection works

The engine detects a fork automatically by checking whether any Level 3 [data-filter-dropdown="level-3-filters-item"] elements inside the active category group have a non-empty data-filter-parent-category attribute. No hardcoded category names. 

const isFork = !!activeGroup.querySelector(
  '[data-filter-dropdown="level-3-filters-item"]' +
  '[data-filter-parent-category]:not([data-filter-parent-category=""])'
);

Level 2 mutual exclusion

When Stump Grinder is checked, Wood Chipper gets muted (opacity 0.5, pointer-events none) and vice versa. Implemented in updateLevel2ForkMuting() which runs after updateLevel2Availability() on every Level 2 checkbox change.

Level 3 item / label visibility

Each Level 3 item has data-filter-parent-category set to the Level 2 slug it belongs to (e.g. carlton-type-stump-grinder). showLevel3() hides items whose parent category doesn't match the currently checked Level 2 value. Label elements work the same way. 

// Items with data-filter-parent-category set in Webflow CMS:
//   carlton-stump-grinder-drive-type-tracks  → data-filter-parent-category="carlton-type-stump-grinder"
//   carlton-wood-chipper-cutting-system-disc → data-filter-parent-category="carlton-type-wood-chipper"

// showLevel3() shows only items matching the active fork parent
item.style.display = (parentCat === selectedParent) ? '' : 'none';

Level 3 availability muting with fork

updateLevel3Availability() skips items that are already fork-hidden (display: none) so fork visibility and availability muting don't conflict.
